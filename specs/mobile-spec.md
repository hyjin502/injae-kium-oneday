# 모바일 반응형 스펙 (Mobile Responsive Spec)

> 상태: **문서 전용 (Spec only)** — 이 문서 작성 시점 기준 `index.html`, `styles.css`, `sections/*`, `tokens.css`는 수정하지 않았습니다. 실제 구현은 이 문서를 승인/확정한 뒤 별도로 진행합니다.
>
> 기준 분석: 2026-07-23 전체 소스 재검토 결과("모바일 반응형 분석 10개 항목")를 바탕으로 작성되었으며, 해당 분석에서 발견된 이슈에 대한 대응 방향을 본 스펙에 반영했습니다.

## 1. 목적 및 범위

- 목적: 현재 PC 전용으로 설계된 랜딩 페이지에 모바일/태블릿 대응 규칙을 정의한다.
- 범위: `index.html` 9개 섹션(Hero ~ Footer) + 향후 추가 예정인 FAQ 섹션.
- 원칙: **PC(1024px 이상)의 기존 레이아웃과 인터랙션은 변경하지 않는다.** 모든 규칙은 `max-width` 기반으로 좁은 화면에만 적용되는 오버라이드로 설계한다.
- 이 문서는 규칙 정의이며, 클래스명/선택자는 현재 코드베이스(`styles.css`, `sections/*.css`) 기준으로 표기했다. 구현 시 그대로 대응된다.

---

## 2. 브레이크포인트

| 구간 | 범위 | 비고 |
|---|---|---|
| 모바일 | 360px ~ 767px | 대표 검수 화면: **390px** (iPhone 12/13/14 기준) |
| 태블릿 | 768px ~ 1023px | 별도 언급 없는 한 PC 레이아웃 준용, 필요한 경우만 개별 규칙 추가 |
| PC | 1024px 이상 | 기존 레이아웃/인터랙션 100% 유지 |

CSS 구현 시 `max-width: 767px`(모바일), `min-width: 768px and max-width: 1023px`(태블릿) 기준을 사용한다. 기존 코드에 이미 존재하는 640px/900px/1024px/1040px/1100px 등 임시 브레이크포인트는 이 표준(360/768/1024)으로 정리·통합이 필요하다 (§8 참고).

## 3. 전역 규칙 (모든 섹션 공통)

| 항목 | 값 | 적용 대상 |
|---|---|---|
| 좌우 여백 (모바일) | 20px | `.container` 및 각 섹션 콘텐츠 wrapper |
| 섹션 상하 간격 (모바일 기본) | 72px | 각 `section-*` 의 `padding` (PC는 기존 120px 유지) |
| 버튼 최소 높이 | 48px | 모든 `button`, CTA, 아코디언 헤더 등 터치 가능 요소 |
| 본문 최소 폰트 | 14px | 본문/설명 텍스트 (제목류 제외) |
| 가로 overflow | 금지 | `body`, `.container`, 각 섹션 루트 — 의도치 않은 가로 스크롤이 생기면 버그로 간주 |
| 가로 스크롤 허용 범위 | "의도된 카드 슬라이더"만 | §5의 5, 6, 7, 9번 섹션에서 명시한 캐러셀/슬라이더 영역에 한함. 그 외 요소(문단, 배너, 그리드)는 반드시 wrap/stack 처리 |
| PC 레이아웃/인터랙션 | 유지 | 1024px 이상에서는 본 문서의 규칙이 적용되지 않음 |

### 3.1 좌우 여백 20px와 기존 `.container` padding(24px)의 관계
현재 `.container { padding: 0 24px }`는 전 구간 공통이다 (`styles.css:41-46`). 모바일 구간에서는 `max-width: 767px`일 때 `20px`로 축소하는 오버라이드가 필요하다 (PC/태블릿은 24px 유지).

### 3.2 섹션 상하 간격 72px과 기존 값의 차이
현재 대부분의 섹션은 `padding: 120px 0`이며, 모바일 축소 규칙이 있는 곳은 section-05(과정 카드)와 section-07(강의장)뿐으로 `64px`로 줄어든다 (`styles.css:905-908`, `styles.css:1140-1142`). 나머지 섹션(Features, Pricing banner, Benefits, Testimonials, Clients, Footer)은 모바일에서도 **120px 그대로**다. 본 스펙 기준으로는 전 섹션이 모바일에서 **72px**로 통일되어야 하며, 기존 64px 값도 72px로 조정이 필요하다.

## 4. 이전 분석과의 연결 (해결해야 할 기존 이슈)

이 스펙의 브레이크포인트/규칙이 실제로 구현되면 아래 항목이 함께 해소되어야 한다.

1. **Section 2 (문제 제기 카드)** — `@media` 부재로 3열 flex가 줄어들지 않아 가로 넘침 위험 → §5-2에서 1열 전환 규칙으로 해결.
2. **Section 3 (지원금 영역)** — `white-space: nowrap` + `@media` 부재로 넘침 위험 → §5-3에서 세로형 전환 + nowrap 해제로 해결.
3. **Section 4 (특징 카드)** — 고정 `grid-template-rows: 157px 157px` → §5-4에서 1열 + auto height로 해결.
4. **Testimonials/Clients 고정폭 카드(316px)** — 360px 기기에서 카드가 넘칠 수 있음 → §5-6, §5-9에서 가로 스크롤 컨테이너화로 흡수(카드 폭은 유지해도 스크롤 영역 안이므로 페이지 전체 가로 스크롤로 번지지 않음).
5. **Hero share 버튼 위치 버그** (`--hero-pad-top`이 모바일 padding 축소 시 갱신되지 않는 문제, `styles.css:74-104` vs `styles.css:246-251`) → §5-1 구현 시 함께 수정 필요.
6. **Hero 아이콘 마퀴 1100px 이하 완전 숨김** → §5-1에서 "비주얼 블록"으로 재정의하여 모바일에 노출.
7. **Lecture facts 4→2열이 480px 이하에서 다시 1열로 바뀌는 문제** (`styles.css:1204-1211`) → §5-8에서 모바일 전 구간(360~767px) 2열 고정으로 수정.

## 5. 섹션별 모바일 규칙

### 5-1. Hero (`.hero`, `.hero-inner`, `.hero-copy`, `.hero-badges`)

- **레이아웃**: `.hero-inner`를 `flex-direction: column`으로 전환 (PC는 좌우 배치 유지).
- **순서**: 텍스트 → CTA → 비주얼.
  - 현재 DOM 순서는 이미 `hero-copy(eyebrow/title/subtitle/cta)` → `hero-badges` 이므로, `flex-direction: column`만 적용하면 순서 요건은 별도 `order` 지정 없이 충족됨.
- **CTA**: `.hero-cta`의 `width: 100%` (기존 모바일 규칙은 `height: 56px`, `padding: 0 24px`만 축소했었음 — 폭 100% 규칙 추가 필요, `styles.css:246-251`). 버튼 높이는 전역 규칙(최소 48px)을 만족하는 56px 유지.
- **비주얼(아이콘 마퀴)**: 기존에는 `@media (max-width: 1100px) { .hero-badges { display: none } }`로 완전히 숨겨졌음. 본 스펙에서는 모바일 구간에서 **숨기지 않고 텍스트/CTA 아래 3번째 블록으로 노출**한다. 다만 PC와 동일한 폭(560px 기준)은 유지 불가하므로 모바일 전용 크기(뱃지 축소, marquee 폭 100%)를 재정의해야 한다. → 상세 치수는 구현 단계에서 별도 확정.
- **PC 마우스 기반 효과 비활성화**: 현재 구현에는 마우스 추적/호버 기반 효과가 없음(`hero-cta-glow`는 호버가 아닌 상시 반복 애니메이션). 따라서 현재는 해당 없음. **향후 마우스 트래킹 효과가 추가될 경우, 터치 디바이스(모바일/태블릿)에서는 반드시 비활성화한다**는 원칙만 문서화해 둔다.
- **공유 버튼 위치 버그**: `--hero-pad-top`이 모바일 `padding` 축소와 함께 갱신되지 않는 문제(§4-5)를 이 섹션 작업 시 함께 수정한다.

### 5-2. 문제 제기 카드 (`.features-cards`, `.feature-card`)

- PC: 3열 flex row 유지.
- 모바일(360~767px): `flex-direction: column`, 카드 `width: 100%`, 카드 간 `gap` 유지(16px 또는 전역 spacing 토큰).
- 태블릿: 별도 규칙 없음(PC 3열 유지). 필요 시 카드 padding만 축소 검토.

### 5-3. 지원금 영역 (`.pricing-banner-inner`, `.pricing-card-row`)

- PC: 좌측(코스 정보) + 우측(가격 정보) 가로 배치 유지.
- 모바일: `.pricing-card-row`를 `flex-direction: column`으로 전환.
- **금액 정보 우선 강조**: 모바일에서는 `.pricing-card-right`(정가/최종 실부담금)를 `.pricing-card-left`(코스 배지/제목)보다 **먼저** 노출한다. 구현 시 DOM 순서 변경 또는 `order` 속성으로 시각적 순서만 조정.
- `.pricing-card-course-text`의 `white-space: nowrap`을 모바일에서 해제(`normal`)하여 긴 코스명이 줄바꿈되도록 한다 (`styles.css:447-454`).
- `.pricing-card-final-price`(현재 40px)는 모바일에서도 시각적으로 가장 큰 요소로 유지하되, 필요 시 폰트 크기만 소폭 축소(예: 32~36px) 검토.

### 5-4. 특징 카드 (`.benefits-grid`, `.benefits-card`) — PC 2열 → 모바일 1열

- PC: `grid-template-columns: repeat(2, 1fr)`, `grid-template-rows: 157px 157px` (고정 높이) 유지.
- 모바일: `grid-template-columns: 1fr` 1열, `grid-template-rows`는 고정값 제거하고 `auto`(콘텐츠에 따라 가변 높이)로 전환. 카드 4개가 세로로 순서대로 쌓임.

### 5-5. 과정 카드 (`.course-grid`, `.course-card`) — 가로 스크롤 슬라이더

- PC: 기존 flex row 레이아웃(2줄 × 3카드) 유지, 1024px 이하에서만 기존처럼 열 전환하던 규칙은 모바일 전용 가로 스크롤 규칙으로 대체됨(태블릿 768~1023px는 기존 1열 스택 규칙 유지 검토 가능).
- 모바일: **가로 스크롤 + scroll snap**.
  - 컨테이너: `overflow-x: auto; overflow-y: hidden; scroll-snap-type: x mandatory;`
  - 카드: `scroll-snap-align: start;`, 카드 폭은 **한 화면에 약 1.1개**가 보이도록 계산 — 예: `flex: 0 0 calc(100vw - 40px)` (좌우 여백 20px×2 반영, 다음 카드가 살짝(약 10~5%) 보이도록 폭을 뷰포트보다 약간 좁게 설정).
  - 스크롤바 숨김: `scrollbar-width: none;` (Firefox), `::-webkit-scrollbar { display: none; }` (WebKit 계열).
  - **자동 슬라이드 사용 안 함** — Section 7(현장 사진)과 달리 JS `setInterval` 기반 autoplay를 적용하지 않는다. 사용자의 수동 스와이프/스크롤로만 이동.
- **열린 결정사항**: 현재 과정 카드는 `.course-grid`가 2개(1행 3장씩, 총 6장)로 나뉘어 있음. 모바일에서 이를 **하나의 연속 가로 스크롤 트랙(6장 전체)으로 병합**할 것을 권장한다.
  - 이유: 두 줄을 각각 별도 가로 스크롤러로 두면 같은 화면에 스크롤 컨테이너가 2개 존재해 사용자가 "다음 줄도 옆으로 넘겨야 한다"는 것을 인지하기 어렵고, 세로 스크롤과 가로 스크롤이 번갈아 필요해 탐색 흐름이 끊긴다. 6장을 한 트랙으로 합치면 한 방향(가로)으로만 자연스럽게 전체 과정을 훑어볼 수 있다.

### 5-6. 고객 후기 (`.testimonials-row`, `.testimonial-card`)

- PC: `flex; justify-content: space-between;` 3카드 고정폭(316px) 가로 배치 유지.
- 모바일: **가로 스크롤, 사용자가 직접 스와이프**.
  - 컨테이너: `overflow-x: auto;` (네이티브 스크롤 — 자동 슬라이드/스냅 강제는 요구되지 않았으므로 우선 자유 스크롤로 정의. 스냅이 필요하면 후속 확정).
  - 카드 폭: 기존 316px 유지 검토(390px 대표 화면 기준 좌우 여백 20px×2 제외 시 350px 가용폭 — 316px 카드는 들어가고 다음 카드가 약간 보임). 360px 최소 폭 기기에서도 카드 자체는 컨테이너 내부에서 스크롤되므로 페이지 전체 가로 스크롤로 번지지 않음(§4-4 대응).
  - 스크롤바 표시 여부는 과정 카드와의 시각적 일관성을 위해 숨김 처리를 권장(§5-5와 동일 처리), 단 사용자 요구사항에 명시되지 않았으므로 확정 전 확인 필요.

### 5-7. 현장 사진 (`.lecture-carousel`, `.lecture-track`, `.lecture-slide`)

- PC: 기존 JS 기반 무한 루프 캐러셀(자동 재생 3.5초 간격 + 이전/다음 버튼) 그대로 유지.
- 모바일: **가로 스크롤 유지, 이미지 비율 유지**(`aspect-ratio: 13/8`, `object-fit: cover` 이미 적용됨, `styles.css:1099-1108`) — 변경 없음.
- **열린 결정사항**: 현재 캐러셀은 `transform: translateX(...)` 기반으로 동작하며 네이티브 `overflow-x` 스크롤이 아니라서, 터치 스와이프 제스처가 자연스럽게 동작하지 않는다(버튼 탭으로만 이동 가능). "PC의 기존 레이아웃과 인터랙션은 유지"라는 전역 원칙에 따라 이번 스펙에서는 **기존 JS 캐러셀 동작을 모바일에서도 그대로 유지**(자동재생 + 화살표 버튼)하는 것으로 정의한다. 네이티브 스와이프 제스처 추가는 별도 개선 과제로 남긴다.

### 5-8. 교육 정보 (`.lecture-facts`, `.lecture-fact`) — PC 4열 → 모바일 2열

- PC: `display: flex;` 4개 항목 가로 배치 유지.
- 모바일(360~767px 전 구간): **2열 고정** — `flex-wrap: wrap; .lecture-fact { flex: 1 0 calc(50% - 12px); }`.
  - 기존 코드에는 `@media (max-width: 480px) { .lecture-fact { flex: 1 0 100%; } }` 규칙이 있어 480px 이하에서 1열로 다시 바뀌는데(`styles.css:1209-1211`), 본 스펙에서는 **모바일 전 구간에서 2열을 유지**하도록 이 480px 규칙을 제거/수정해야 한다(§4-7).

### 5-9. 강사 카드 (`.clients-cards`, `.client-card`)

- PC: `flex; justify-content: space-between;` 3카드 고정폭(316px) 가로 배치 유지.
- **모바일 방식 결정: 세로 1열 스택**을 권장.
- **선택 이유**:
  1. 카드 수가 3개로 적어, 가로 스크롤의 이점(많은 항목을 훑어보는 탐색성)이 크지 않다.
  2. 강사 카드는 프로필/뱃지/소개 문단/대표 강의처까지 정보 밀도가 높은 카드다. 가로 스크롤 카드로 두면 한 번에 하나씩만 보이고 폭도 제한(316px)되어 문단(`.client-card__desc`, 최대 3줄 이상)이 답답하게 느껴질 수 있다. 1열로 풀 폭을 주면 문단이 더 여유 있게 읽힌다.
  3. 반면 Section 6(고객 후기)은 카드당 정보량이 짧은 인용문 + 이름 한 줄뿐이라 스와이프로 가볍게 훑기에 적합해 가로 스크롤을 유지했다 — 즉 "카드 수" 자체보다 "카드당 정보 밀도"가 두 섹션의 처리 방식을 가르는 기준이다.
  4. 신뢰도를 다루는 섹션(강사 경력/이력)은 사용자가 한 번에 놓치지 않고 다 읽는 것이 전환에 더 유리하다고 판단.
- 구현: `flex-direction: column;` 또는 `display: grid; grid-template-columns: 1fr;`, 카드 `width: 100%`.

## 8. 브레이크포인트 표준화 메모

현재 코드베이스에 산재한 임시 브레이크포인트(`640px`, `900px`, `1024px`, `1040px`, `1100px`)는 이 문서의 표준(360 / 768 / 1024)에 맞춰 정리가 필요하다. 구현 단계에서 섹션별로 다음과 같이 통합하는 것을 권장한다:

- `640px`, `900px` 등 모바일 내부 세부 조정용 브레이크포인트 → `767px` 이하 규칙 안에서 필요한 경우 보조 분기로만 남기고, 열 수/구조 전환 같은 큰 레이아웃 변경은 반드시 `360~767 / 768~1023 / 1024+` 3단 기준을 따른다.
- `1040px`, `1100px`(Testimonials wrap, Hero badges 숨김)은 태블릿 상한(1023px)과 PC 하한(1024px) 경계에 맞춰 재검토.
